Observations & Risks

Architecture notes

  • Clean, conventional Vue 3 + FastAPI split — easy to extend.
  • Shared state handled well via singleton composables (filters, auth, i18n).
  • i18n scaffolding (en/ja) already present — useful for D2.
  • A purchase-orders endpoint exists — a starting point for the restocking feature (R2).

Risks & gaps

  • No database. All data is static JSON loaded in-memory; writes (tasks, POs) don't persist across restarts.
  • API base URL hard-coded to localhost in api.js — no environment config.
  • Reports page shows defects on inspection — primary remediation target (R1).
  • No automated tests in the delivered code — covered by R3.
  • Client calls /api/tasks the backend doesn't appear to fully implement — needs verification.
